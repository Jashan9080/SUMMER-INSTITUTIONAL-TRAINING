# DAY-8
Network Troubleshooting
## Date
10 July 2026
### Objective
To understand common network troubleshooting techniques, diagnose network connectivity issues, and learn the basic configuration of networking devices such as routers and switches.
### Topics Covered
### 1. Network Troubleshooting Commands
Learned the purpose and usage of common Windows network diagnostic commands:
- **ipconfig** – Displays IP configuration details.
- **ipconfig /release** – Releases the current IP address.
- **ipconfig /renew** – Requests a new IP address from the DHCP server.
- **ipconfig /flushdns** – Clears the DNS cache.
- **ping** – Tests network connectivity with another device or server.
- **tracert** – Displays the path taken by packets to reach the destination.
- **nslookup** – Checks DNS name resolution.
- **netstat** – Displays active network connections and ports.
These commands are useful for identifying and troubleshooting network-related problems.
### 2. Resolving IP Conflicts and DHCP Errors
Studied common causes of IP conflicts and DHCP-related issues.
- Understood that an **IP conflict** occurs when two devices on the same network are assigned the same IP address.
- Learned that DHCP errors occur when a device is unable to obtain a valid IP address automatically from the DHCP server.
- Discussed basic troubleshooting methods such as renewing the IP address, checking network cables, restarting the router, and verifying DHCP settings.
### 3. APIPA (Automatic Private IP Addressing)
Learned about **APIPA**, which automatically assigns an IP address in the range **169.254.x.x** when a DHCP server is unavailable.
Understood that APIPA indicates:
- DHCP server is not responding.
- Network cable may be disconnected.
- Router or switch may not be functioning properly.
- The system has limited network connectivity.
### 4. Basic Router and Switch Configuration
Studied the basic functions of routers and switches.
Learned about:
- Connecting network cables correctly.
- Basic router settings.
- LAN and WAN ports.
- Restarting and resetting a router.
- Difference between rebooting and factory resetting a router.
- Role of switches in connecting multiple devices within a Local Area Network (LAN).
### 5. Wi-Fi Connectivity Issues
Learned the common reasons behind Wi-Fi connection problems.
Topics discussed:
- Weak Wi-Fi signal strength.
- Physical obstacles affecting signals.
- Wireless interference from nearby devices.
- Incorrect Wi-Fi password.
- Router placement.
- Network congestion.
- Restarting the router to resolve connectivity issues.
### 6. Firewall and Antivirus
Studied how firewall and antivirus software affect network communication.
Learned that:
- A firewall protects the system by blocking unauthorized access.
- Incorrect firewall settings may block websites or network services.
- Antivirus software can sometimes restrict internet access if suspicious activity is detected.
- Proper configuration helps maintain both security and connectivity.
