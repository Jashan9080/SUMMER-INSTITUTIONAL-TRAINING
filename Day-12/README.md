### Theory
### HTML Semantic Tags
Semantic tags describe the purpose of the content they contain. They make webpages easier to understand for developers, browsers, and search engines.
### Common Semantic Tags
#### `<header>`
Defines the introductory section or heading of a webpage.
#### `<nav>`
Contains navigation links to different pages or sections.
#### `<section>`
Groups related content into separate sections.
#### `<article>`
Represents independent content such as blog posts or news articles.
#### `<footer>`
Defines the bottom section of a webpage containing copyright or contact information.
### CSS (Cascading Style Sheets)
CSS is used to design and style HTML webpages. It controls colors, fonts, spacing, layouts, and overall appearance while keeping design separate from content.
### Types of CSS
#### 1. Inline CSS
- Written inside the HTML element using the `style` attribute.
- Applies style to only one element.
#### Example
html
<h1 style="color:blue;">Welcome</h1>

#### 2. Internal CSS
- Written inside the `<style>` tag in the `<head>` section.
- Styles all matching elements within one webpage.
### Example
html
<style>
h1{
color:blue;
}
</style>
### 3. External CSS
- Written in a separate `.css` file.
- Linked using the `<link>` tag.
- Used for styling multiple webpages.
### Advantages of CSS
- Makes webpages attractive.
- Separates design from content.
- Reduces code repetition.
- Easy to maintain.
- Reusable across multiple webpages.
- Improves website consistency.
- Faster webpage modifications.
- Better user experience.
### Disadvantages of CSS
- Browser compatibility issues may occur.
- Large CSS files become difficult to manage.
- Debugging complex styles can be time-consuming.
- Requires knowledge of selectors and properties.
